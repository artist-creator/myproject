# myproject User Story
1. Application Registration
As a new user, I want to register using my name, email, and password so that I can create a personal movie tracking account.
Example: The app allows users to enter their details and tap the Sign Up button to create an account.
Why: This enables personalized movie tracking and data storage.

2. Application Login
As a registered user, I want to log in using my email and password so that I can access my saved movie list.
Example: The login screen verifies credentials and navigates the user to the home screen.
Why: This ensures secure and personalized access.

3. Error Feedback on Signup and Login
As a user, I want to receive error messages if I enter incorrect details so that I understand what went wrong.
Example: The app shows “Invalid email or password” when login fails.
Why: Clear feedback improves usability and reduces confusion.

4. Store User Data
As a user, I want my login details and movie list to persist so that I don’t lose my data between sessions.
Example: The app stores user data locally so it remains after reopening the app.
Why: Persistence improves user experience and reliability.

1. View Movie Overview
As a user, I want to see my “Watch Next” and “Already Watched” sections on the home screen so that I can quickly manage my movies.
Example: The home screen shows categorized movie cards.
Why: This allows efficient navigation and tracking.

2. Quick Navigation
As a user, I want to quickly add a new movie from the home screen so that I can update my watchlist easily.
Example: A floating action button (+) opens the Add Movie screen.
Why: Reduces steps and improves usability.

3. Movie Summary at a Glance
As a user, I want to see movie title, year, and IMDb rating in the list so that I can evaluate movies quickly.
Example: Each movie card displays name, year, and rating.
Why: Helps users prioritize what to watch next.

1. View Detailed Information
As a user, I want to view detailed information about a selected movie so that I can decide whether to watch it.
Example: The detail screen shows title, year, IMDb rating, cast, and description.
Why: Detailed insights support informed decisions.

2. Mark Movie as Watched
As a user, I want to mark a movie as “Already Watched” so that I can update my watch progress.
Example: A button changes movie status from Watch Next to Watched.
Why: Keeps the tracking system accurate.

3. Edit Movie Information
As a user, I want to edit movie details so that I can correct or update information.
Example: Users can modify rating or description.
Why: Ensures flexibility and accurate tracking.

1. Persistent Watch List
As a user, I want my watchlist to remain saved even after closing the app so that I don’t lose my movie data.
Example: Movies remain stored after reopening the app.
Why: Ensures seamless experience.

2. Save Movie Status
As a user, I want my watched/unwatched status to persist so that my progress remains accurate.
Example: Movies marked as watched stay in the Watched category.
Why: Maintains progress tracking.

3. Save User Preferences
As a user, I want my preferences (like sort order or theme) saved so that the app remembers my settings.
Example: Dark mode stays enabled after restarting.
Why: Improves personalization.

1. Fetch IMDb Ratings Automatically
As a user, I want the app to fetch IMDb ratings automatically so that I get accurate and up-to-date information.
Example: The app integrates a movie API to auto-fill rating and year.
Why: Saves time and ensures reliable data.

2. Fetch Cast Information
As a user, I want movie cast details fetched from an external API so that I get complete information.
Example: Selecting a movie automatically loads actor names and description.
Why: Enhances content richness.

1. Access Settings
As a user, I want to access settings from any screen so that I can adjust preferences easily.
Example: A settings icon is available in the top-right corner.
Why: Improves accessibility.

2. Organized Settings Categories
As a user, I want settings categorized (Profile, Notifications, Theme) so that I can find options quickly.
Example: Settings page shows grouped options.
Why: Logical structure improves user experience.
